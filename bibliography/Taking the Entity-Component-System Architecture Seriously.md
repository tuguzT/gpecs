# Taking the Entity-Component-System Architecture Seriously

A conference talk that examines what an engine would look like if [ECS](/notes/ecs/ecs.md) were treated as the primary
architecture for everything: data layout, scheduling, ergonomics, and higher-level features such as relationships and indexes.

# Component

Particular aspect of some [entity](entity.md) with **data** needed to model such aspect.

Key concept of component could be demonstrated as follows:

![Component](/notes/attachments/entity-component.png)

Usually, there could be only **one** instance of component for an entity.

Entity-to-component relationship are managed by [sparse sets](/notes/topics/data-structures/sparse-set.md) or [archetypes](archetype.md).
